Operating System as IDE

Last updated: Tuesday, Dec 13th, 2020

Integrated Development Environment, or IDE, are supposed to be a environment that provides useful tools and utilities for development (of software, and other works). The environment could be a single piece of monolithic software that people run, like say MS VSCode, PyCharm, Eclipse, etc. But there is no reason Linux cannot be a IDE. Unfortunately, the word IDE has now been hijacked to mean monolithic software and that Linux is a IDE feels wrong.

I would argue, knowing way around in Linux makes it so much better as an IDE than anything out there. An IDE has some very important parts like a text editor, a file manager, source control management, etc. So does Linux and it excels in it. Want a text editor, choose from hundreds of option. Want a file manager, hundreds of option. Same for others, hundreds of tools for any conceivable task, even for something that is generally not considered as a part of IDE. The OS acts as a glue, putting it all together. And customize to all your liking. I believe that was how computers were meant to be used and how old-school developers used it.

I would go as far as to argue that using a "IDE" on top of your operating system is, in fact, detrimental. The developer now has two environment taking their cognitive space. Using the operating system makes it more coherent and has less congnitive load. Here are a few tips and tricks to use Linux as a IDE. Most of my interaction with the operating system happens with the keyboards. So, this is a very keyboard oriented advice.

imv - X11/Wayland Image Viewer

For images, I have found imv to be suitable for all my needs. And most importantly it uses vi-keybindings, auto-reload and minimal UI. Most of my image related tasks comes down to writing a piece of code that generated a image and looking at the image if it is correct, or looking at the pixels. Image viewers are mostly mouse-related programs. But imv provides good vi-like keybindings for working and therefore, it lets me keep my hands on the keyboards. Here are some keybinding:
key description
h, j, k, l Pan image
i, o, rZoom in/out, reset
c-r Rotate image
f, dToggle fullscreen/overlay
x, qClose/Quit
imv loads any config located ~/.config/imv/config.
