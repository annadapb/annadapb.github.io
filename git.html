git diff - See difference between two files

Without any flags or options, the git diff command will show the differences in any file that are in the current working directory and the staging area. The output of the command is a patch file, similar to the diff Linux/Unix command. Any difference can be then patched with the patch command.

  • git diff
    Show the diff of all files in the working directory and staging area.
  • git diff --cached (or --staged)
    See the diff of all files in the staging area and the most recent commit.
  • git diff c1ab343 4ab43ed
    Diff of all files in commit c1ab343 and commit 4ab43ed
  • git diff c1ab343 4ab43ed filename
    git diff c1ab343:filename 4ab43ed:filename
    Same as above but only for filename.
  • git diff HEAD^ [filename]
    git diff HEAD~3 HEAD [filename]
    Also works with relative references.
  • git diff @~4.. > patchfile
    patch patchfile     Save the diff as a patch file and apply the patch.

Note: (1) Old revisions must comes first for the + and - in the patch file to be applied properly by patch util. (2) Git needs two commits or the stage for diff and will accept a range of commits but it will igonre the rest!
